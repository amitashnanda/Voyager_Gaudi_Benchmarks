================================================================================
METHOD NOTES
================================================================================
Sensitivity Analysis Method: PMPQ
  PMPQ (Pruning-based Mixed-Precision Quantization) computes
  layer sensitivity as the accuracy drop on BoolQ TRAIN split
  when a fixed percentage of the smallest-magnitude weights in that
  layer are set to zero. All other layers remain unchanged.
  sensitivity[i] = baseline_accuracy - pruned_accuracy.
  Layers with larger accuracy drops are more sensitive and receive
  higher bit-widths in mixed-precision quantization.
  Calibration data: Phase 1 training split of BoolQ.

Quantization: FAKE (Simulated)
  Each weight: FP32 -> round to [8, 8, 4] levels -> dequantize
  -> stored back as FP32. Captures rounding error numerically
  but uses no fewer bytes in memory. Standard approach used by
  OmniQuant, AffineQuant, and similar papers for measuring accuracy
  impact without hardware deployment.

Group-Wise Quantization:
  group_size=128: every 128 consecutive weights
  share one scale factor. Same approach as GPTQ. Provides finer
  precision than a single global scale per layer.

Dataset: BoolQ Validation Set
  Binary reading comprehension benchmark (yes/no questions).
  Evaluation: log-likelihood scoring of 'yes' and 'no' tokens
  given the passage + question prompt. Model picks the token
  with the higher log-likelihood.
  Metrics: Accuracy, Precision, Recall, F1 (positive class = True).

Clustering: percentile with 3 groups
  Sensitivity scores clustered to assign bit-widths:
  Cluster 0 (rank 0, bits=8): 7 layers, mean sensitivity=0.0098
  Cluster 1 (rank 1, bits=8): 7 layers, mean sensitivity=-0.0033
  Cluster 2 (rank 2, bits=4): 8 layers, mean sensitivity=-0.0134
