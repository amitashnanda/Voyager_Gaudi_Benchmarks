================================================================================
METHOD NOTES
================================================================================
Sensitivity Analysis Method: PMPQ
  PMPQ uses normalized Frobenius norm of weight matrices.
  sensitivity(layer_i) = mean(||W||_F / sqrt(|W|)) over
  all Linear modules in layer i.
  Higher score -> larger weight magnitude -> more sensitive
  -> needs higher bit-width to preserve accuracy.
  Lower score  -> smaller magnitude -> less sensitive.

Quantization: FAKE (Simulated)
  Each weight: FP32 -> round to [16, 8, 4] levels -> dequantize
  -> stored back as FP32. Captures rounding error numerically
  but uses no fewer bytes. Standard approach for measuring
  accuracy impact without hardware deployment.

Group-Wise Quantization:
  group_size=128: every 128 consecutive weights
  share one scale factor. Same approach as GPTQ.

Dataset: WikiText-2 Test Set
  Language modeling benchmark.
  Metric: Perplexity (lower is better).
  Additional metric: throughput in tokens/s.
  Note: Phase 1 calibration used train split.
        Phase 2 evaluation always uses test split.

Clustering: percentile with 3 groups
  Cluster 0 (rank 0, bits=16): 7 layers, mean sensitivity=2.189888
  Cluster 1 (rank 1, bits=8): 7 layers, mean sensitivity=0.827283
  Cluster 2 (rank 2, bits=4): 8 layers, mean sensitivity=0.543486
