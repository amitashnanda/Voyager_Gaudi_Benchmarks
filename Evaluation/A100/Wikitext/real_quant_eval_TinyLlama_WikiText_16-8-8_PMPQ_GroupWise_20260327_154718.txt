================================================================================
METHOD NOTES
================================================================================
Sensitivity Analysis Method: PMPQ
  PMPQ uses normalized Frobenius norm of weight matrices.
  sensitivity(layer_i) = mean(||W||_F / sqrt(|W|)) over
  all Linear modules in layer i.
  Higher score -> larger weight magnitude -> more sensitive
  -> needs higher bit-width (FP16) to preserve perplexity.
  Lower score  -> smaller magnitude -> can use INT8.

Quantization: REAL (weights stored in FP16 or INT8 on GPU)
  FP16 layers: weight stored as torch.float16.
               A100 Tensor Cores run FP16 matmul natively.
  INT8 layers: weight stored as torch.int8 (1 byte per weight).
               On each forward pass: int8 weights dequantized
               to FP16 using per-group scale, then FP16 matmul.
               This is weight-only quantization (same as GPTQ).
               INT8 is purely a storage format -- computation
               runs in FP16 on A100 Tensor Cores.

Group-Wise Quantization:
  group_size=128: every 128 consecutive weights
  share one FP16 scale factor. Same approach as GPTQ.

Dataset: WikiText-2 Test Set
  Language modeling benchmark.
  Metric: Perplexity (lower is better).
  Additional metric: throughput in tokens/s.
  Note: Phase 1 calibration used train split (512 tokens).
        Phase 2 evaluation always uses test split.

Clustering: kmeans with 3 groups
  Cluster 1 (rank 0, bits=16 [FP16]): 1 layers, mean sensitivity=7.459869
  Cluster 2 (rank 1, bits=8 [INT8]): 2 layers, mean sensitivity=1.786510
  Cluster 0 (rank 2, bits=8 [INT8]): 19 layers, mean sensitivity=0.759747
